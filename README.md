# USDA FruitAndVegetables Data Source

QuantConnect data source integration for USDA FruitAndVegetables.

## Overview

USDA ERS retail price estimates for 150+ commonly consumed fresh and processed fruits and vegetables. Includes price per pound/pint and normalized price per edible cup equivalent.

Each produce + form combination is its own data series (symbol). Data is stored per series in `alternative/usda/fruitandvegetables/{series}.csv` with the date in the first column (for example, `alternative/usda/fruitandvegetables/apples_fresh.csv`).


## Data Characteristics

| Property | Value |
|----------|-------|
| **Data Source ID** | 0 |
| **Format** | CSV |
| **Resolution** | Daily |
| **Timezone** | Utc |
| **Requires Mapping** | False |
| **Sparse Data** | True |

## Data Processing

The `DataProcessing` workflow downloads (or reads) USDA ERS Fruit & Vegetable Prices `.xlsx` workbooks and emits per-series CSV files in Lean's data folder structure.

Each XLSX workbook from USDA ERS contains pricing data organized by produce type and form (Fresh, Canned, Frozen, Dried, Juice).

### Configuration Architecture

DataProcessing uses **QuantConnect's Config.Get system** for all configuration. The `config.json` file in the `DataProcessing/` directory serves as the configuration source that QuantConnect's `Config.Get` reads from.

**How it works:**
1. `config.json` contains key-value pairs for configuration
2. Python code loads `Config` from QuantConnect via `CLRImports.py` (using pythonnet/CLR bridge)
3. All configuration is accessed via `Config.Get(key)` - never by reading JSON directly
4. This ensures consistency with QuantConnect's cloud environment where config comes from environment variables

**Authentication:** This dataset does not require an API key. If USDA requires one in the future, set `vendor-auth-token` in `config.json` or via environment variable - the downloader will send it as an `X-Api-Key` header.

**Output directory:** Uses `data-folder` from Config.Get (typically `../../../Data/`) and can be overridden by `temp-output-directory`.

**Date range:** DataProcessing requires `process-start-date` and `process-end-date` via Config.Get and fails fast if either is missing.

### Run (Python, recommended)

1. Download the USDA `.xlsx` file(s) for the year(s) you want to process.
2. Ensure `pandas` + `openpyxl` are available (supported by QuantConnect's Python environment).
3. Configure `DataProcessing/config.json` with your desired settings:

**Minimal config (only required fields):**
```json
{
  "process-start-date": "20130101",
  "process-end-date": "20230101"
}
```

**Full config with all options:**
```json
{
  "process-start-date": "20130101",
  "process-end-date": "20230101",
  "temp-output-directory": "./output",
  "http-timeout-seconds": 120,
  "max-retries": 5,
  "rate-limit-requests": 10,
  "rate-limit-seconds": 1.1,
  "usda-fruitandvegetables-max-xlsx-downloads": 0,
  "normalize-cup-equivalent-unit": true
}
```

| Key | Required | Default | Description |
|-----|----------|---------|-------------|
| `process-start-date` | ✅ | - | Start date (yyyyMMdd) |
| `process-end-date` | ✅ | - | End date (yyyyMMdd) |
| `temp-output-directory` | | `data-folder` | Output directory |
| `http-timeout-seconds` | | 60 | HTTP request timeout |
| `max-retries` | | 5 | Max HTTP retry attempts |
| `rate-limit-requests` | | 10 | Max requests per window |
| `rate-limit-seconds` | | 1.1 | Rate limit window |
| `usda-fruitandvegetables-max-xlsx-downloads` | | 0 (unlimited) | Max files to download |
| `normalize-cup-equivalent-unit` | | true | Normalize fluid_ounces to pints |
| `vendor-auth-token` | | "" (empty) | API authentication token (sent as X-Api-Key header) |

4. Run from the `DataProcessing` directory:

```bash
cd DataProcessing && python3 process.py
```

### Outputs

- CSV files: `alternative/usda/fruitandvegetables/{series}.csv`
- Series manifest: `docs/series-manifest.json` (generated by `datasources/USDAFruitAndVegetables/scripts/generate_series_manifest.py`)
- Helper class: `USDAFruitAndVegetables.Symbols.cs` (generated by `datasources/USDAFruitAndVegetables/scripts/generate_series_catalog.py`)
- Schema artifacts: `DataProcessing/src/core/schema.py` and `USDAFruitAndVegetables.Schema.cs`
  (generated by `datasources/USDAFruitAndVegetables/scripts/generate_schema_artifacts.py`)

CI runs the manifest/catalog scripts to keep generated files in sync; avoid hand-editing generated files.
If you run the series scripts locally, install `jinja2`.

```bash
dotnet build
```

## Testing

```bash
dotnet test tests/Tests.csproj
python3 -m unittest discover -s DataProcessing/tests -t DataProcessing
```

Downloader tests make live HTTP requests. To enable them, either:
- Set the environment variable `USDA_RUN_LIVE_TESTS=1`, or
- Set `usda-run-live-tests=1` in `Config.Get`
